# Day 05 — Conditions

## Exercises

**5a.** Создай константу `age`. Используй `if`, чтобы проверить — можно ли арендовать машину (обычно от 21 года). Выведи сообщение если можно.

**5b.** Создай две константы — названия любимых блюд. Сравни их через `<` и `>`, выведи какое идёт первым по алфавиту.

**5c.** Создай массив `cart` из 3 товаров. Добавь четвёртый через `.append()`. Если товаров стало больше 3 — удали самый первый через `if` и `remove(at:)`.

**5d.** Создай константу `enteredCode` со значением какого-то кода. Сравни через `==` с правильным кодом `"1234"`, выведи результат. Затем то же самое через `!=`.

**5e.** Создай переменную `comment` со значением пустой строки. Через `if` и `.isEmpty` замени её на `"Без комментария"`.

**5f.** Создай константу `outsideTemp = 5`. Через `if`/`else` выведи `"Тепло"` или `"Холодно"` (порог подбери сам).

**5g.** Создай константу `examScore`. Через `if`/`else if`/`else` выведи оценку: `"A"` (90+), `"B"` (75-89), `"C"` (60-74), `"F"` (ниже).

**5h.** Создай `isStudent: Bool` и `age: Int`. Через `&&` проверь — положена ли скидка студентам младше 25 лет.

**5i.** Создай `isAdmin: Bool` и `isModerator: Bool`. Через `||` проверь — есть ли у пользователя доступ к панели управления.

**5j.** Создай `hasSubscription: Bool`, `isTrialActive: Bool`, `isBanned: Bool`. Через `&&`/`||` с явными скобками проверь — может ли пользователь смотреть контент (подписка ИЛИ триал, но только если не забанен).

**5k.** Создай enum `DeliveryStatus` с case `processing`, `shipped`, `delivered`, `returned`. Через `switch` выведи сообщение для каждого статуса.

**5l.** Создай константу `userInput` типа `String` с произвольным значением. Через `switch` с `default` обработай 3 конкретных варианта и всё остальное.

**5m.** Создай константу `stage = 3` (этап обратного отсчёта запуска ракеты). Через `switch` и `fallthrough` выведи все этапы от текущего до старта (`"Зажигание!"`, `"3…"`, `"2…"`, `"1…"`, `"Старт!"`).

**5n.** Создай `isWeekend: Bool`. Через тернарный оператор выведи `"Отдыхаем"` или `"Работаем"`.

**5o.** Создай массив `notifications`. Через тернарный оператор и `.isEmpty` выведи `"Нет уведомлений"` либо количество через interpolation.

## 🏆 Challenge 1 — Кинотеатр

Система расчёта цены билета:
- До 12 лет — бесплатно
- 12-17 — детская цена
- 18-64 — взрослая цена
- 65+ — пенсионная скидка

Используй `if`/`else if`/`else`, выведи итоговую цену через string interpolation.

## 🏆 Challenge 2 — Торговый автомат

Создай enum `VendingItem` с 5 товарами (вода, чипсы, шоколад и т.д.). Через `switch` для каждого товара выведи его цену. Добавь `default` для несуществующего товара.

## 🏆 Challenge 3 — Контроль доступа в игре

Создай систему проверки: игрок может войти в закрытую зону, если он `isVIP` или (`hasKey` и `questCompleted`). Используй `&&`/`||` с явными скобками. Выведи через тернарный оператор статус `"Доступ открыт"` / `"Доступ закрыт"`.

## 🏆🔄 Mixed Challenge — Система регистрации на мероприятие

**a.** Создай enum `Category` с типами билетов: `standard`, `vip`, `staff`.

**b.** Создай словарь `ticketPrices: [String: Int]` с ценами для каждой категории (строковые ключи, совпадающие с названиями case).

**c.** Создай множество `checkedInGuests: Set<String>` — список тех, кто уже прошёл регистрацию.

**d.** Создай переменную `guestName: String` и `guestCategory: Category`.

**e.** Через `if` и `.contains()` проверь — не зарегистрирован ли гость уже. Если зарегистрирован — выведи `"Уже отмечен!"`.

**f.** Через `switch` по `guestCategory` выведи приветственное сообщение, разное для каждой категории.

**g.** Через тернарный оператор определи скидку: `isEarlyBird ? 10 : 0` (создай `Bool isEarlyBird` сам).

**h.** Выведи итоговую сводку через string interpolation: имя, категория, цена с учётом скидки, статус регистрации.
